## Goal

Ship a demo-ready MVP in 24 hours with clear scope control and execution speed.

<WorkflowStep step={1} title="4-Hour Planning Block">
Break down the idea into must-have, should-have, and nice-to-have features. Lock the must-have set before coding.
</WorkflowStep>

<WorkflowStep step={2} title="12-Hour Build Block">
Use Cursor/Copilot for component and API scaffolding. Use Claude Code for larger refactors and bug-fix passes.
</WorkflowStep>

<WorkflowStep step={3} title="4-Hour Polish Block">
Add deployment, bug fixes, and a short scripted demo flow.

<PromptBlock agent="USER">
Create a 90-second demo script that explains the problem, solution, and one key technical innovation.
</PromptBlock>
</WorkflowStep>

<WorkflowStep step={4} title="4-Hour Buffer">
Reserve time for integration failures, flaky dependencies, and deployment surprises.
</WorkflowStep>

## Expected Output

A stable demo, clear pitch, and a codebase that can be continued after the hackathon.
