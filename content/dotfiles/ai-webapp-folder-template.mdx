## Overview
This template standardizes app structure so feature work does not stall on folder decisions.

## Recommended Structure
```txt
app/
components/
content/
lib/
scripts/
tests/
```

## Rules
- Keep product logic in `lib/`.
- Keep page-level composition in `app/`.
- Keep reusable UI in `components/`.
- Keep quality gates in `scripts/` and `tests/`.

## Setup Checklist
1. Add SEO defaults in root layout metadata.
2. Add content validation script early.
3. Add at least one list page and one dynamic detail page.
4. Add tests for filtering/searching behavior before scaling content.
