## Overview
This config set focuses on safe defaults, short feedback loops, and repeatable quality checks while working with coding agents.

## Alias Examples
```sh
alias qc='npm run validate:content && npm run lint && npm run test'
alias gs='git status -sb'
alias gd='git --no-pager diff'
```

## Session Rules
- Never run destructive git commands without explicit approval.
- Prefer narrow, focused diffs over large rewrites.
- Validate after every meaningful change set.
